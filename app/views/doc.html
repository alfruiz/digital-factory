API DOCUMENTATION

METHOD URL DESCRIPTION BODY RESPONSE
GET /api/numbers Get the saved information inside the token
Notes:
* token need to be send on HEADERS as Authorization		 STATUS: 200
{ numbers: [array_of_number] }
POST /api/token Create Token, with name
Notes:
* < data > is a String 		{ name: < data > } STATUS: 200
{ token: 'newtoken' }
PUT /api/numbers update content of token
Notes:
* token need to be send on HEADERS as Authorization
* < data > is an Array 		{ numbers: < data > } STATUS: 200
{ msg: 'numbers are upload to token: < token >' }

STEPS

  1. Create a token (please keep it, you have to send it after the exam) - POST
  2. Update the info of your token - PUT
  3. Get data of your token - GET
